# 🏛️ **中国历史知识系统——功能需求设计文档**

## 1. 项目整体定位

一款集 **历史时间轴、人物百科、疆域与地理演变、古今地名对照、历史遗迹知识库、AI 历史助手** 于一体的中国历史学习系统。
 技术栈：**Java + Spring Boot + MySQL + MyBatis + Redis + Vue**

------

# 2. 功能模块设计

------

# 🌟 **A. 时间轴浏览系统**

## A1. 全历史时间轴

- 支持「史前 → 夏商周 → …… → 现代中国」完整时间线。
- 时间轴可缩放（年 → 十年 → 百年）。
- 按朝代、事件、人物生卒进行定位。

## A2. 朝代导航

- 按顺序展示中国历史各朝代。
- 点开朝代可查看：
  - 开国灭亡年份
  - 都城
  - 著名君主
  - 重要事件
  - 社会制度/文化背景

## A3. 重大历史事件标记

- 时间轴事件分类（政治、军事、文化、科技等）。
- 每个事件含：
  - 事件简介
  - 背景原因
  - 相关人物
  - 历史影响

------

# 🌟 **B. 历史人物百科**

## B1. 人物数据管理

字段信息（示例）：

- 姓名、字/号
- 所属朝代
- 生卒年（自动挂载主时间轴）
- 人物简介、主要事迹
- 相关事件列表
- 图片

## B2. 人物关系图谱

可视化展示人物之间的关系：

- 亲属（父子、兄弟、夫妻）
- 师承
- 政治 / 军事关系（君臣、敌对等）

## B3. 人物生平时间线

- 根据人物生卒自动绘制人物时间轴。
- 可展示关键人生节点（出生、官职变迁、战役、成就等）。

------

# 🌟 **C. 地理与疆域演变模块**

## C1. 动态疆域地图

- 不同时期的国家疆域展示（使用 GeoJSON）。
- 时间滑块切换朝代疆域。
- 显示都城、主要州/郡、战役地点。

## C2. 古今地名对照

支持查询古代地名 → 映射现代地名
 例如：

- “长安” → “陕西西安”
- “汴梁” → “河南开封”

## C3. 历史遗迹标注

包含：

- 遗址介绍
- 对应的历史事件
- 地理位置（用于地图展示）

------

# 🌟 **D. AI 历史助手**

## D1. 历史问答

- 支持用户查询任何历史事件、人物、朝代背景等。
- 能根据用户提问返回简明或扩展说明。

## D2. 数据库联动

AI 回答可自动引用系统数据库的数据，如：

- 人物详情
- 事件关联
- 朝代时间信息
   并在回答中提供可点击跳转。

## D3. 生成型功能

- 总结朝代历史
- 生成人物年表
- 对比历史事件

------

# 🌟 **E. 用户系统**

## E1. 用户基础功能

- 注册、登录、JWT 认证
- 修改个人信息
- 重置密码

## E2. 阅读记录

- 自动记录用户浏览的历史事件、人物、朝代
- 方便回溯学习进度

------

# 🌟 **F. 性能与系统设计**

## F1. Redis 缓存

- 热门朝代、事件、人物缓存
- 时间轴数据缓存
- 地名对照缓存

## F2. 搜索系统

- 支持人物、事件、地名的全文搜索
- 可接入 ElasticSearch

## F3. 数据规模预估

- 朝代数据：数十条
- 历史事件：几千条
- 人物：上万条
- 人物关系：10 万级
- 地名：几千条
